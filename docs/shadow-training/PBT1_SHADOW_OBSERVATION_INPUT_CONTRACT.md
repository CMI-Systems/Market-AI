# PBT-1 Shadow Observation Input Contract

Date: 2026-06-16

## Approved PBT-1 Mode

AICC mode: DASHBOARD_MODE

Brain mode: SHADOW_OBSERVATION_MODE

Output authority: NON_AUTHORITATIVE

PBT-1 is an observation-only preparation phase. It must not activate Shadow Training, the Shadow Trainer, Brain Learning, live learning, autonomous trading, order execution, candidate promotion, or production deployment.

## Approved Input Fields/Classes

PBT-1 may consume only existing locked AICC inputs:

- validated Alpaca quote/snapshot data
- validated Alpaca candle/OHLCV data
- symbol
- timeframe
- provider
- sourceType
- timestamp/provider timestamp
- receivedAt/dataAge where available
- sessionState and marketOpen
- validationStatus and qualityLabel
- availability, stale, partial, cached, delayed, unavailable states
- simulated/generated flags
- provenance and warnings
- Tactical derived state
- Behavioral market-state derived state
- Failsafe state
- Consensus/Regime/Narrative state where already generated by existing engines
- journal entries owned by the authenticated operator
- replay sessions owned by the authenticated operator
- operator history derived from owned journal/replay/dataset records
- dataset record metadata
- dataset validation results
- shadow readiness results
- dataset governance results
- provider/backend/system status metadata

## Rejected Inputs

PBT-1 must reject:

- Webull
- unknown provider/source as trusted input
- simulated/generated market data as certified input
- news/headlines/sentiment/catalysts
- macro data
- market breadth
- global multi-market data beyond existing US equities scope
- options data
- broker account data
- positions/orders/buying power
- API keys, cookies, tokens, service-role keys, credentials
- inferred psychology
- new behavioral telemetry fields
- clinical/emotional diagnosis fields
- cross-operator aggregates
- placeholder/static fixtures as evidence
- new schema fields not approved through a schema-change phase

## Brain-Specific Consumers

### Tactical Brain

Consumes approved market context only: symbol, quote/candles, OHLCV, timestamp, timeframe, trend/momentum/volatility/liquidity context, provider/source/session/provenance/validation metadata.

No news, Webull, broker-account, macro, breadth, or generated/simulated input is approved.

### Behavioral Brain

Consumes existing market-behavior context and, for operator behavioral review, owned journal/replay/operator-history evidence. Operator emotion, confidence, discipline, and bias may only come from explicit operator evidence already present in the architecture.

No inferred psychology or clinical claims are approved.

### Failsafe Brain

Consumes provenance, validation, provider/session/freshness state, simulated/generated flags, Tactical/Behavioral/Consensus/Regime status, conflicts, warnings, provider/backend status.

Failsafe retains blocking authority.

## Output Authority Restrictions

PBT-1 brain outputs:

- are diagnostic only
- are not operator-facing as validated guidance
- do not affect Command Center verdicts
- do not affect Signals
- do not affect Alerts
- do not affect operator-visible Consensus/Regime
- do not trigger trades
- do not modify positions
- do not create broker instructions
- do not suppress Failsafe warnings
- do not automatically modify datasets outside approved capture rules

## Persistence Restrictions

No PBT-1 observation persistence is currently authorized.

Any future observation persistence must:

- use approved dataset/observation schema
- preserve operator ownership
- preserve source timestamps
- preserve provenance and validation metadata
- include run ID, dataset ID, schema version, validation version, code/brain/config version where practical
- store outputs as non-authoritative observations
- avoid secrets, credentials, cookies, and session tokens
- avoid production writes unless separately approved

## No-Learning Guarantee

PBT-1 must not:

- update weights
- update thresholds
- update rules
- create model artifacts
- train a candidate
- mark data training-ready
- mark data training-eligible outside governance gates
- promote or deploy candidates

## No-Live-Write Guarantee

PBT-1 must not write to:

- live brain source files
- active runtime imports
- active provider configuration
- production environment settings
- live broker/order systems
- Command Center verdict state
- Signals/Alerts/Consensus/Regime authoritative state

## No-Schema-Expansion Guarantee

PBT-1 accepts no new input fields. Any new field, provider, telemetry category, or persistence table requires a separate approved schema-change phase.

## PBT-1 Exit Criteria

PBT-1 preparation may exit only when:

- authenticated product-surface QA is complete or explicitly accepted as a formal limitation
- observation logging design is approved
- schema/versioning gaps are resolved or accepted
- no training, Shadow Trainer, Brain Learning, or promotion path is active
- Failsafe blocking authority remains preserved
- shadow outputs remain non-authoritative
- all rejected inputs remain blocked
